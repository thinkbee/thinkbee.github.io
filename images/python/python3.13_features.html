Python 3.13

제미나이 도움을 받아 Python 3.13의 주요 특징 요약하고 정리했다.

  • Python 3.13은 2024년 10월에 첫 정식 릴리스
  • https://www.python.org/downloads/release/python-3130/

주요 특징

다음은 Python 3.13의 주요 특징이다.

1. 향상된 대화형 인터프리터 (REPL)

  • 다중 라인 편집 및 히스토리: REPL 에서 여러 줄의 코드를 작성하고 편집하기가 훨씬 쉬워졌다. 위/아래 화살표 키로 이전 명령을 탐색할 때 전체 코드 블록을 가져오고 편집할 수 있다.
  • 스마트 붙여넣기: 코드 블록을 REPL에 붙여넣을 때 자동으로 인식하고 올바르게 처리한다.
  • 바로 가기: help(), exit(), quit()과 같은 REPL 명령을 함수 호출 없이 help, exit, quit으로 바로 사용할 수 있다.
  • 컬러 지원: 트레이스백 및 doctest 출력에 컬러가 기본으로 적용되어 가독성이 향상되었다.
  • 화면 지우기: clear 명령으로 화면을 지울 수 있습니다.

2. GIL (Global Interpreter Lock) 제거 (실험적)

  • Free-threaded CPython (PEP 703): 가장 큰 변화 중 하나로, GIL을 비활성화하고 스레드 간 병렬 실행을 가능하게 하는 실험적 지원이 추가되었습니다. 이는 멀티코어 시스템에서 스레드를 사용하는 프로그램의 성능을 크게 향상시킬 잠재력이 있습니다.
  • 별도 빌드 필요: 이 기능은 현재 실험적이며 기본적으로 활성화되어 있지 않습니다. 이 기능을 사용하려면 일반적으로 python3.13t와 같은 별도의 실행 파일을 사용해야 합니다.

3. JIT (Just-In-Time) 컴파일러 (실험적)

  • PEP 744: 기본적인 JIT 컴파일러가 추가되었다. 기본적으로 비활성화되어 있고, 향후 릴리스에서 성능 향상을 기대할 수 있다.

4. 개선된 오류 메시지

  • 더 유용한 제안: 잘못된 키워드 인자 전달 등 일반적인 실수에 대해 더 유용하고 구체적인 오류 메시지를 제공합니다.
  • 컬러 트레이스백: 기본적으로 오류 트레이스백에 색상이 적용되어 디버깅이 더 쉬워졌습니다.

5. 정적 타입 힌트 개선

  • typing.TypeIs (PEP 742): 타입 좁히기 (type narrowing)를 위한 새로운 타입 힌트입니다.
  • TypedDict의 읽기 전용 항목 (PEP 705): TypedDict 내에서 읽기 전용 항목을 표시할 수 있습니다.
  • 타입 시스템을 통한 Deprecation 표시 (PEP 702): 타입 시스템을 사용하여 더 이상 사용되지 않는 항목을 표시할 수 있습니다.
  • 타입 매개변수의 기본값 (PEP 696): 타입 매개변수에 기본값을 설정할 수 있습니다.

6. copy.replace() 함수 추가

  • 불변 객체 변경: collections.namedtuple, dataclasses.dataclass, datetime.datetime 등 여러 내장 타입 및 __replace__() 메서드를 정의하는 모든 클래스에 대해 copy.replace() 함수가 추가되어 불변 객체를 쉽게 변경할 수 있습니다.

7. Docstring 최적화

  • 메모리 사용량 감소: Docstring의 선행 들여쓰기가 제거되어 메모리 사용량과 .pyc 파일 크기가 줄어듭니다.

8. 기타 변경 사항

  • dbm.sqlite3 모듈: 이제 기본 dbm 백엔드입니다.
  • random 모듈의 CLI: random 모듈에 명령줄 인터페이스가 추가되었습니다.
  • os 모듈 개선: Linux의 타이머 알림 파일 디스크립터 작업 및 Windows의 chmod() 관련 함수 등 os 모듈에 여러 새로운 함수가 추가되었습니다.
  • 모바일 플랫폼 지원: iOS 및 Android가 Tier 3 지원 플랫폼으로 추가되었습니다.

Python 3.13 주요 특징 코드

Python 3.13의 일부 새로운 기능은 사용 방식이 크게 다르지 않지만, 몇 가지 특징을 보여주는 코드는 다음과 같습니다.

1. 향상된 REPL (직접 실행해야 체감 가능)

REPL의 변화는 실제 Python 3.13 환경에서 경험해야 합니다. 하지만 간단한 예시로 설명할 수 있습니다.

exit(), help() 함수가 아닌 ‘exit’ 또는 ‘help’ 명령으로 사용 할 수 있다.

2. copy.replace() (불변 객체 변경)

namedtuple이나 dataclass와 같은 불변 객체의 특정 필드만 변경하여 새로운 객체를 생성한다.

불변 객체 내부적으로 __replace__() 메서드를 구현하는 객체들을 위한 함수.

  • collections.namedtuple: 튜플이지만 이름 기반으로 항목에 접근하고 _replace() 메서드를 가집니다.
  • dataclasses.dataclass (frozen=True 일 때): 불변 데이터 클래스를 만들 때 사용됩니다.
  • datetime.datetime 등 표준 라이브러리의 불변 객체들.

class 에서 불변 dataclass 에서도 사용할 수 있다.

3. 타입 힌트 개선 (TypeIs 예시)

TypeIs는 런타임에 특정 조건에 따라 타입이 좁혀지는 것을 명확히 할 때 사용된다.

다음은 TypeIs를 사용하여 리스트가 특정 타입의 요소만 포함하고 있음을 확인하는 함수를 정의하고 있다.

4. warnings.deprecated() 데코레이터

더 이상 사용되지 않는 함수나 클래스를 표시하고 사용자에게 경고를 제공할 때 사용됩니다.
